## Original Post

## Idea

Instead of MCP tools dumping 50KB+ JSON into context, the LLM writes a tiny extraction script, runs it in a sandbox, and only the compact output enters context. 65-99% context savings.

## Source
- Gist: https://gist.github.com/chenhunghan/27ef2fece7329ac7b88f815ce0c0fbb1
- Origin: copilotclaw#158

## Why it matters for us
- Our GitHub MCP calls are noisy (list_issues, search_code return walls of JSON)
- We're burning premium quota — this pattern would stretch context budgets significantly
- Maps to our existing `.github/scripts/` pattern: instead of `gh api | jq` chains, the LLM generates the jq/Python filter and runs it locally

## Implementation path
- Sandbox: `RestrictedPython` for our Python-based scripts
- The gist includes copy-paste prompt to add it to any MCP server

## Next step
Apply when we build custom MCP tooling or want to tighten how GitHub API calls consume context.
